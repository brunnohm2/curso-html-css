Capítulo 22 - Iframe

Aula 1 - O iframe ainda pode ser usado?

O que significa iframe?

É um Frame Inline "Em português forçado: quadro em linha".

A tag utilizada é <iframe src="" frameborder=""></iframe>.

Existem sites que não aceitam o uso de iframe. Um exemplo de site que não aceita o uso do iframe é o Google.

Aula 2 - Configurando iframes

Todo o conteúdo que estiver dentro da tag <iframe src="" frameborder=""></iframe> será exibido caso a tag retornar um erro.

Exemplo: 

<iframe src="url" frameborder="0">
    <a href="url">link</a>
</iframe>

Parâmetros

1) width e height tem como padrão os valores 350px de altura e 150px de altura respectivamente.

Observação importante: caso definir um valor de altura na tag iframe e outro valor de altura em CSS, as definições em CSS são soberanas ao valor atribuído na tag iframe.

2) scrolling aceita 3 valores:

  • auto: adiciona barra de rolagem automaticamente;
  • yes: barra de rolagem ativa permanentemente;
  • no: barra de rolagem desativada.

Existem comportamentos diferente entre os tipos de navegadores com o parâmetro scrolling.

3) frameborder aceita somente 2 valores, as configurações de borda se aplica em CSS:

  • 0: sem borda;
  • qualquer outro valor: com borda.

Aula 3 - Conteúdo local no iframe

Por padrão o iframe o display é de valor inline. Se for necessário centralizar o iframe no centro da tela, será necessário alterar o display para inline no CSS.

Indexação de conteúdo

O uso de iframes em páginas torna-se inviável para a indexação de conteúdo feito pelos motores de busca. Quando o motor de busca começa a ler o site e chega no iframe e pula todo o conteúdo contido no arquivo carregado pelo mesmo.

Aula 4 - Navegação no iframe

Criando links

Para criar links e carregar o arquivo no iframe desejado é necessário atribuir um valor no parâmetro name e no link adicionar o valor no parâmetro target.

<a href="url" target="frame">Carregar</a>
<iframe name="frame" src=""></iframe>

Aula 5 - Conteúdo no iframe por código

Carregando Documentos

srcdoc pode inserir html dentro para que o iframe exiba o próprio conteúdo.

<iframe name="frame" srcdoc="<h1>Texto</h1>"></iframe>

Nesse caso será necessário utilizar aspas simples no lugar de aspas duplas se for preciso.

Aula 6 - Inconvenientes do iframe

Indexação de motores de busca

O uso de iframes pode impedir a indexação do conteúdo quando o site for analisado pelo motor de busca. Assim podendo fazer com que haja a desqualificação no seu sistema de buscas.

Acessibilidade do Site

Softwares para deficientes visuais tem dificuldade com a interpretação do uso de iframes, assim dificultando a usabilidade do site.

Problema com a navegação do site

Alguns navegadores não conseguem interpretar se dever avançar ou voltar o conteúdo principal, ou do iframe. E o mesmo ocorre com o parâmetro target de valor _blank.

Responsividade

Caso o conteúdo do site carregado pelo iframe não for responsivo, causara possíveis problemas na exibição de seu conteúdo.

Segurança

O site carregado pode coletar dados a partir do iframe, permitindo a abertura para técnicas de invasão, como Cross-Site scripting (XSS).

Aula 7 - Tornando iframes mais seguros

Parâmetro sandbox

Valor sandbox: adiciona nível de segurança mais alto que não deixa coletar nenhum tipo de dado no iframe.

Valor allow-same-origin: permitir conteúdo de mesma origem.

Valor allow-forms: permite formulários.

Valor allow-scripts: permite scripts.

Parâmetro referrerpolicy

Vem com o valor padrão no-referrer-when-downgrade

Para tornar mais limitado a captura de dados é necessário alterar o valor para no-referrer.

Aula 3 - Métodos GET e POST para formulários

Em HTML5 existem dois métodos de envio de formulário, o método get e post.

Por padrao o método get é o método padrao de envio do formulário.

Quando usar os métodos GET e POST?

GET - Quando os dados requisitados não for sensíveis e podem ser compartilháveis. Esse método não pode passar 3.000 bytes ou aproximadamente 3.000 letras.

POST - Quando os dados for sensíveis como senhas, dados de cartões de créditos, arquivos grandes.

Aula 4 - Criando caixas de textos e de senha

Exercício 25 > form002.html e contato.php.

O código abaixo está com o método GET, mesmo que o input de senha não seja exibida na tela, após o envio do formulário, os caracteres da senha serão exibidos na url do navegador.

<form action="contato.php" method="GET">
    <label for="i-nome">Nome</label>
    <input type="text" name="nome" id="i-nome">
    
    <label for="i-senha">Senha</label>
    <input type="password" name="senha" id="i-senha">
    
    <input type="submit" value="Enviar">
    <input type="reset" value="Limpar">
</form>

Abaixo está no formato de envio POST, assim os dados de senha não ficará explicito na url do navegador após o envio do formulário, porém ainda é possível obter os dados de senha através do DevTools do navegador.

<form action="contato.php" method="POST">
    <label for="i-nome">Nome</label>
    <input type="text" name="nome" id="i-nome">
    
    <label for="i-senha">Senha</label>
    <input type="password" name="senha" id="i-senha">
    
    <input type="submit" value="Enviar">
    <input type="reset" value="Limpar">
</form>

Atributo required, comando de preenchimento obrigatório.

<form action="contato.php" method="POST">
    <label for="i-nome">Nome</label>
    <input type="text" name="nome" id="i-nome" required>
    
    <label for="i-senha">Senha</label>
    <input type="password" name="senha" id="i-senha" required>
    
    <input type="submit" value="Enviar">

Atributo minlength e maxlength, determina o número mínimo e máximo de caracteres respectivamente.

<form action="contato.php" method="POST">
    <label for="i-nome">Nome</label>
    <input type="text" name="nome" id="i-nome" required minlength="5" maxlength="15">
    
    <label for="i-senha">Senha</label>
    <input type="password" name="senha" id="i-senha" required minlength="8" maxlength="20">
    
    <input type="submit" value="Enviar">
    <input type="reset" value="Limpar">
</form>

Atributo size, aumenta o tamanho da caixa com a quantidade de caracteres correspondente.

<form action="contato.php" method="POST">
